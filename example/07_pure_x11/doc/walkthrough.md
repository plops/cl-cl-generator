## Overview

This guide provides a comprehensive, step-by-step walkthrough for developers. It covers setting up prerequisites, executing the code generator, inspecting generated source code, running demo applications, executing unit test suites, creating custom widgets, building custom MUV applications, and extending the X11 core protocol layer.

## Step 1: Prerequisites

Ensure your environment meets the following requirements:

1. **Common Lisp Environment:** [SBCL (Steel Bank Common Lisp)](https://www.sbcl.org/) version 2.0 or newer.
2. **Package Manager:** Quicklisp installed and configured in your SBCL init file (`~/.sbclrc`).
3. **Display Server:** An active X11 display server (`DISPLAY` variable set, e.g., `:0` or `:99`).
4. **Build Tools:** Git installed for automatic version stamping in generated headers.

## Step 7: Creating a Custom Widget (`PROGRESS-BAR`)

To add a custom widget type to the toolkit, register its rendering function with `register-widget`.

### Code Example: `PROGRESS-BAR` Renderer
```lisp
(in-package :pure-x11-gen)

(register-widget "PROGRESS-BAR"
  (lambda (w-struct focused pressed hovered)
    (declare (ignore focused pressed hovered))
    (let* ((x (widget-x w-struct))
           (y (widget-y w-struct))
           (w (widget-w w-struct))
           (h (widget-h w-struct))
           (props (widget-props w-struct))
           (value (or (getf props :value) 0.0)) ; 0.0 to 1.0
           (fill-w (round (* (- w 4) (max 0.0 (min 1.0 value))))))
      ;; Background frame
      (poly-fill-rectangle (list (list (+ x 2) (+ y 2) (- w 4) (- h 4))) :gc *gc-light*)
      (draw-bevel x y w h :style :sunken :bevel-width 2)
      ;; Filled progress meter
      (when (> fill-w 0)
        (poly-fill-rectangle (list (list (+ x 2) (+ y 2) fill-w (- h 4))) :gc *gc-text*)))))
```

## Step 8: Building a Custom MUV Application

Here is a complete, minimal single-file MUV application:

```lisp
(defpackage :my-counter-app
  (:use :cl :pure-x11-gen))
(in-package :my-counter-app)

;; 1. Model State
(defstruct state (count 0))

;; 2. Pure Update
(defun update-fn (st msg)
  (case (car msg)
    (:inc (make-state :count (1+ (state-count st))))
    (:dec (make-state :count (1- (state-count st))))
    (t st)))

;; 3. Pure View (Virtual DOM)
(defun view-fn (w h st)
  `(panel :name :root :x 0 :y 0 :w ,w :h ,h
     (vbox :name :main :x 10 :y 10 :w ,(- w 20) :h ,(- h 20) :padding 5 :spacing 10
       (label :name :lbl :text ,(format nil "Counter Value: ~a" (state-count st))
              :glue (:natural 24 :stretch 0 :shrink 0))
       (hbox :name :btns :glue (:natural 32 :stretch 0 :shrink 0) :spacing 10
         (button :name :b-inc :text "Add" :msg (:inc) :glue (:natural 100 :stretch 1 :shrink 0))
         (button :name :b-dec :text "Sub" :msg (:dec) :glue (:natural 100 :stretch 1 :shrink 0))))))

;; 4. Application Entry Point
(defun run ()
  (run-gui #'update-fn #'view-fn (make-state)))
```

## Step 9: Extending the X11 Protocol Layer

To add a new X11 core request (e.g., `SetInputFocus`, Opcode 42):

### 1. Add Request Spec to `02_x11_spec.lisp`
Open `02_x11_spec.lisp` and add the new specification into `*x11-requests*`:

```lisp
    ;; SetInputFocus (Opcode 42)
    (:name set-input-focus
     :doc "Set input focus to specified window."
     :params (focus-window &key (revert-to 1) (time 0))
     :packet ((card8 42)             ; Opcode
              (card8 revert-to)      ; Revert-to policy (1 = RevertToPointerRoot)
              (card16 3)             ; Length in 4-byte words
              (card32 focus-window)  ; Window ID
              (card32 time)))        ; Timestamp
```

### 2. Export Symbol in `generate.lisp`
In `generate.lisp`, add `#:set-input-focus` to the `:export` list of `pure-x11-gen`.

### 3. Re-run Generator & Test
```bash
sbcl --load generate.lisp
```

`write-source` will emit the new function `set-input-focus` into `source/x11-core.lisp` automatically:

```lisp
(defun set-input-focus (focus-window &key (revert-to 1) (time 0))
  "Set input focus to specified window."
  (progn
    (with-packet
      (card8 42)
      (card8 revert-to)
      (card16 3)
      (card32 focus-window)
      (card32 time))))
```
