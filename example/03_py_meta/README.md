# Example 03: cl-py-generator Meta-Generator

This folder contains a Common Lisp to Common Lisp meta-generator for the `cl-py-generator` transpiler's `py.lisp` compiler. 

By defining helper builders that process boilerplate at generation-time (such as parentheses formats, relational lists, binary-only operations, assignments, strings, and logical operators), we reduce duplication while emitting clean, highly repetitive but fully correct output.

## Files

- [gen.lisp](file:///home/kiel/stage/cl-cl-generator/example/03_py_meta/gen.lisp): The meta-generator template and helpers.
- [py.lisp](file:///home/kiel/stage/cl-cl-generator/example/03_py_meta/py.lisp): The fully expanded, formatted Lisp code generated by `gen.lisp`.
- [run-tests.sh](file:///home/kiel/stage/cl-cl-generator/example/03_py_meta/run-tests.sh): Portable script to run the transpiler tests using our generated `py.lisp`.

## Instructions

### 1. Generating `py.lisp`
To run the generator and output the new `py.lisp` source file inside this directory:
```bash
sbcl --load example/03_py_meta/gen.lisp --eval "(quit)"
```

### 2. Running `cl-py-generator` Unit Tests
To run the full unit test suite from `cl-py-generator` using the generated `py.lisp`:
```bash
./example/03_py_meta/run-tests.sh
```
This script dynamically registers the local `cl-py-generator` package path, quickloads it, swaps the compiler implementation with our generated `py.lisp`, and runs all 144 unit tests.

## New DSL Syntax & Features Reference

The transpiler has been refactored to streamline S-expressions, remove deprecated constructs, and introduce advanced Python features.

### 1. Core Syntax & keyword Renames
- **`do` â†’ `body`**: The construct for indenting/grouping block-level statements is now named `body`.
- **`cl-py-generator:do0` â†’ `progn`**: Simplified top-level/non-indented block statement grouping.
- **`dictionary` â†’ `dict*`**: Formats keyword arguments into a Python `dict(...)` constructor (e.g., `(dict* :a 1 :b 2)` â†’ `dict(a=1, b=2)`).
- **`~` â†’ `lognot`**: Exported unary bitwise negation operator (e.g. `(lognot x)` â†’ `~x`).
- **`return` consolidation**: The `return_` construct has been removed. A single `return` statement now handles empty return `(return)`, single value `(return x)`, or multiple values `(return (ntuple x y))`.

### 2. Streamlined Imports
- **`import`**: Now variadic and supports aliases directly (e.g., `(import sys (np numpy))` â†’ `import sys\nimport numpy as np`).
- **`import-from`**: Imports symbols from a module (e.g., `(import-from math sin cos)` â†’ `from math import sin, cos`). The older `imports` and `imports-from` wrapper constructs have been removed.

### 3. Advanced String Handling
The `string` construct now accepts optional keyword modifiers to produce different Python string types:
- **Bytes Literal**: `(string :bytes "data")` â†’ `b"data"`
- **Triple-Quoted Block**: `(string :triple "block")` â†’ `"""block"""`
- **Raw string**: `(string :raw "data")` â†’ `r"data"`
- **f-string**: `(string :f "val={x}")` â†’ `f"val={x}"`
- **Combinations**: Modifiers can be combined, such as `(string :raw :triple "raw block")` â†’ `r"""raw block"""`.
- **F-string Auto-detection**: Bare strings at the block level now automatically analyze and format expression interpolation (e.g., `"The value is {x + 1}"` â†’ `f"The value is {x + 1}"`).

### 4. Raw Code & Statement Insertion
- **`raw`**: Inserts arbitrary raw text directly into the generated Python file (e.g., `(raw "df = pd.read_csv('data.csv')")`).
- **`assert`**: Emits an assert statement with optional message: `(assert condition)` or `(assert condition "Error message")`.
- **`yield` / `yield-from`**: Native Python yield expressions: `(yield x)` or `(yield-from gen)`.
- **`decorator` / `decorated`**: Native support for decorators and decorated functions.

### 5. Precedence and Associativity Formatting
The `paren*` precedence-aware grouping utility has been updated to accept `:side l` or `:side r` arguments, ensuring operators of identical precedence (like `+` and `-`) are parenthesized correctly according to Python's left/right associativity properties.
