# Walkthrough - Improving gen02.lisp

We have completed the refactoring and improvement of [gen02.lisp](file:///workspace/src/cl-cl-generator/example/04_tui_cockpit/gen02.lisp). The generation template has been cleaned up by leveraging the generation-time features of `cl-cl-generator`.

## Changes Made
1. **Dynamic Slot Generation for [cockpit-model](file:///workspace/src/cl-cl-generator/example/04_tui_cockpit/gen02.lisp#L61-L83):**
   - Extracted slot specifications to an inline list.
   - Spliced them dynamically using `,@(loop for ...)` to construct the `defclass` structure automatically. This avoids copy-pasting `:initarg` and `:accessor` names for all 20 slots.
2. **Abstracted Process File Parsers:**
   - Defined `make-pid-parser` to eliminate duplicated `with-open-file` logic for simple file retrieval (such as `oom_score` and `comm`).
   - Refactored `parse-pid-oom` and `parse-pid-name` definitions inside the template to use the new generator helper.
3. **Abstracted TUI Key Binding Checks:**
   - Defined `match-key` to clean up the `cond` branch keys inside the `tui:update-message` method.
   - Handled key-matching for case-insensitivity of letters (`q`/`Q`, `h`/`H`, `t`/`T`) as well as optional keyword mapping (e.g. `:escape` or `:f1`).
